# Hex Self-Improver Rubric

## Event Types

- `error`: command/tool/process failure that blocked progress
- `learning`: correction, better method, or validated optimization
- `feature`: requested capability not currently available

## Priority

- `low`: nice-to-have
- `medium`: useful recurring improvement
- `high`: cost/time/safety impact
- `critical`: security/data loss or severe breakage risk

## Promotion Guide

Promote when at least one is true:
1. repeated 2+ times
2. saves significant time on common workflow
3. affects safety/security
4. changes strategy direction
